The DBMS must provide sufficient languages and interfaces for each user group to communicate database queries and updates. Database Languages are used to create and maintain a computer database. There is a wide range of database languages, such as Oracle, MySQL, MS Access, dBase, FoxPro, etc. SQL statements widely found in Oracle and MS Access can be classified as Data Definition Language (DDL), Data Control Language (DCL), and Data Control Language (DML).

♦ Data Definition Language (DDL)

It is a language that allows users to define their data and their relationship to other types of data. It is mainly used to create files, databases, data dictionaries and tables within databases. This is often used to define the structure of each table, the values associated with each attribute, the integrity constraints, the protection and authorization information for each table, and the physical storage configuration of each table on the disk. The following table gives a summary of the use of DDL statements in SQL.

S.No need and usage the SQL DDL statement
1 creat schema abjects CREAT
2 alter schema abjects ALTER
3 delete schema abjects DROP
4 reneme schema abjects RENEME

♦ Data Manipulation Language (DML)

It is a language that provides a set of operations to support basic data manipulation of the data held in the databases. Allows users to insert, update, delete, and retrieve data from the database. The part of DML that involves the retrieval of data is called the query language. The following table gives an overview of the use of DML statements in SQL:

S.No need and usage the SQL DML statement
1 remove rows from tables or views DELETE
2 add new rows of data into table or view INSERT
3 retrieve data from one or more tables SELECT
4 change column values in existing rows of a table or view UPDATE

Data Control Language (DCL)

DCL statements control access to data and databases using statements such as GRANT and REVOKE. A right can either be given to the User with the help of a GRANT declaration. The assigned privileges may be SELECT, ALTER, DELETE, EXECUTE, INSERT, INDEX, etc. In addition to granting privileges, it can also be revoked (taken back) by using the REVOKE command. The following table gives an overview of the use of DCL statements in SQL:

S.No need and usage the SQL DML statement
1 grant and take away priviliges and roles GRANT
2 add a comment to the data dictionary COMMENTT

In reality, the language of data definition and data manipulation is not two different languages. They simply form parts of a single database language, such as the Structured Query Language ( SQL). SQL is a mixture of DDL and DML, as well as constraint definition and schema evaluation statements.
