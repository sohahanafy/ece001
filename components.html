♦ Components

A database consists of:

Database structure

Conceived as a single entity consisting of a collection of interrelated files.

Repository

Includes the rules under which users can access the data and the rules under which the data is organized, i.e. the metadata.

End users

Who have access to the database.

Interfaces

Which are the various ways that end-users can access data via display terminals, remote terminal phone links, touch screens, bar code readers, voice commands, touchpads, graphics selection, keyboard, or mouse menu selections, command keys or card scanning. The interface is also the form in which the information is provided to the end-user, e.g. a graph, a printed report, graphical display.

Application programs

Used to perform main operations on data, i.e. create, alter, delete, and retrieve. Application programs also combine data in a meaningful way to produce reports

Database Management System (DBMS)

A collection of programs that manage the database structure and interpret the repository rules as well as manage user access in multi-user systems.

Computer Aided Software Engineering Tools (CASE tools)

are also known as part of the database system. These are automated tools to assist in the design of systems and databases.

Database Administrators (DBA)

Manage the DBMS and are responsible for the organization 's overall information resources. In a later section, you will look more fully at the role of the DBA.

System Developers

Include system analysts and programmers. You 're going to look at the function of each of these in other modules. Both use CASE tools to help develop application programs. Components of a Database
