A database can be defined as a shared collection of interrelated data designed to meet the organization's varied information needs. (The McFadden and the Hoffer).

♦ Properties

A database is

Integral

Previously distinct data files have been logically organized to eliminate or reduce redundancy and facilitate data access

Shared

All eligible participants of an organization have access to the same data for use in a variety of activities.

Interrelated

Structured in a manner that is logically meaningful to the organization.

♦ Advantages

There are a range of benefits to the approach to the database. These are discussed in more depth in the topic of Database Systems. There are some of the advantages of a database system

Data management

The DBMS provides a mechanism for facilitating the management of the data held by the organization.

Data independence

Data is stored in a database system independently of the application programs that use it. Changes to the data or how it is stored will not necessarily affect the programs that use the data.

Data consistency

The database system is designed to minimize redundancy. Ideally, data items are stored only in one location. When data updates are made, adjustments should only be made to this one spot. This significantly improves the consistency of the database test results.

Data sharing

Data collection in a central data repository and managed by DBMS enables the sharing of data by several users and applications.

Increased productivity

File structures are already established in the database structure. New application programs that use these frameworks can then be created. This results in a shorter period of growth.

Reduced maintenance

A range of tools for data maintenance has already been built into the majority of DBMS. Tools can include report generators, query languages, type generators, code tests, data dictionaries, and maintenance logs.
